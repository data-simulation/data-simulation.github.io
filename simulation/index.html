Obtención de datos


Registro en video de la plaza Luis Braille de Alicante realizado el 26 de Nomviembre entre las 5 y las 6 de la tarde con un teléfono movil.


Procesamiento de las imágenes mediante el algoritmo YOLO. Se detectan distintos agentes y categorias: person, bench, car, truck y bird.





Los arboles de la plaza dificultan el análisis de las imágenes por lo que se opta por completarlo con grabaciones a pie de calle.


La categoría BIRD es la que se detecta con menor precisión. YOLO detecta el conjunto pero no los individuos.


Al no tener buenos datos de los pájaros, se simulan según las condiciones del algoritmo Flocks, herds and schools: A distributed behavioral model (1987) de Craig Reynolds: separación. alineación y cohesión.




La simulación se pone a prueba con los datos del entorno real. A las reglas del algoritmo se añaden condiciones de dimensión del entorno, presencia y movimiento de personas y vehículos, ruido y tiempo para que el comportamiento se ajuste al de la plaza Luis Braille.


Esta simulación se representa tridimensionalmente para permitir un mejor entendimiento del espacio por parte de humanos.


La base de datos recopila los datos reales y los simulados para representar todas las dimensiones del entorno.




Aunque el modelo 3D es el que mejor entienden las personas, hay dimensiones que no alcanzan a representar, como la evolución en el tiempo.


La gran cantidad de datos manejados satura el modelo 3D, por lo que se plantean otros métodos de representación que traducen las condiciones a mapas, gráficas y matrices de datos. Del cruzamiento de datos con algoritmos de clusterizado se pueden obtener otros nuevos, como categorías de uso o temporalidad de las actividades.



Toda la información se codifica y se comprime por dos motivos. Las GPUs trabajan de forma óptima con matrices cuadradas y con la intención de codificar para homogeneizar todas las diferentes plazas y sus condiciones (tipo, tamaño, usos) bajo un lenguaje común.




Esta codificación está pensada para los ordenadores, no para las personas, por lo que no se puede entender a simple vista sin decodificar.




Esta codificación alberga todas las características que definen cada espacio urbano para comparar todas las distintas plazas y sus relaciones y que el algoritmo aprenda de la base de datos.
